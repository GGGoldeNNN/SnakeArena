# 系统与平衡性

> 综合多本书籍关于游戏经济系统、数值平衡、难度平衡的理论知识。

---

## 平衡性的目的（来自《体验引擎》）

### 公平性的平衡
所有玩家有公平获胜的机会。对称阵营（如象棋）天生公平，非对称阵营需要精细设计。

### 深度的平衡
没有单一最优策略（即不存在「必胜法」）。多种策略都可行，玩家可以根据情况选择。

### 其他平衡类型
- 不同角色/兵种之间的平衡
- 不同战术路线的平衡
- 不同阶段（前期/中期/后期）的平衡

---

## 策略退化（Strategy Degradation）（来自《体验引擎》）

一种策略明显优于其他所有策略时，其他策略就「退化」了——虽然它们在理论上存在，但实际无人使用。

- **可选策略数量的误区**：重要的不是策略的数量，而是这些策略是否**都真正有效**
- **平衡的对象是策略而非数字**：数据平衡只是手段，策略多样性才是目的

---

## 平衡和技巧（来自《体验引擎》）

- **平衡的对象**：需要平衡的是不同技巧水平下的体验
- **是否需要平衡性**：不是所有游戏都需要完美平衡（如合作游戏、纯剧情游戏）
- **平衡的方法**：
  1. 数据收集（胜率、使用率等）
  2. 针对性调整
  3. 测试验证
  4. 重复迭代

---

## 博弈论在游戏中的应用（来自《体验引擎》）

### 核心概念
- **纳什均衡**：所有玩家都没有动机单方面改变自己策略的状态
- **混合策略**：随机化选择策略以防止被对手预测
- **石头剪子布循环克制**：A 克 B、B 克 C、C 克 A 的平衡模型

### 心理战
游戏中加入心理博弈层面，让玩家之间的互动超越纯策略层面。
- 案例分析：《现代战争2》中的心理战术

## 系统和平衡透镜（来自《游戏设计艺术》）

> **透镜 #46：系统** — 游戏的各个部分如何相互作用？改变一个部分会如何影响其他部分？
> **透镜 #49：平衡** — 游戏中是否有某种策略明显优于其他？所有选项是否都值得选择？
> **透镜 #50：公平** — 玩家觉得游戏公平吗？失败是因为技巧不足还是系统不公？
> **透镜 #55：反馈** — 玩家如何知道自己的表现？反馈是否及时清晰？

### 系统思维
- 游戏是一个相互关联的系统
- 改变任何一个元素都会产生涟漪效应
- 好的平衡需要理解系统的整体行为而非局部

## 动态的游戏系统（来自《游戏设计梦工厂》）

### 游戏作为一个系统
- 游戏是各部分相互关联、相互影响的系统
- 改变任何一个部分都会产生连锁反应
- 系统动态：系统的行为模式随时间变化

### 平衡性检查清单（来自《游戏设计梦工厂》）
- 是否有某种策略明显优于其他？
- 是否有策略完全没人用？
- 新手是否有公平的获胜机会？
- 高手是否仍然有挑战性？
- 游戏在不同人数/场景下都平衡吗？

### 功能性、完成度、平衡性三者关系
- **功能性**：游戏是否按设计运行（无 bug）
- **完成度**：内容是否齐全（无缺失）
- **平衡性**：游戏是否公平有趣
- 三者需同时满足，缺一不可

---

## 乐趣的要素（来自《游戏设计梦工厂》）

### 改进玩家的选择
- 好的选择系统 = 有趣的游戏
- 选择要有意义、有后果、有变化

### 调节和平滑
- 难度曲线的精细调整
- 避免难度骤升或骤降

### 人类对抗机器
- 击败系统本身带来的满足感
- 系统应有「聪明」的响应

## 从新书增加的系统与平衡知识

### 成长曲线设计（来自《平衡掌控者：游戏数值战斗设计》）

**三要素**：
1. **初始值**：起点能力，决定前期体验
2. **成长速度**：每级增量，决定成长手感
3. **上限值**：终点能力，决定后期天花板

**4种体验目的的曲线选择**：

| 曲线类型 | 特征 | 体验效果 | 适用场景 |
|---------|------|---------|---------|
| **前期快速型** | 初期成长快、后期趋缓 | 前期爽快，后期稳定 | 手游/休闲游戏 |
| **后期爆发型** | 前期成长慢、后期加速 | 厚积薄发，后期强势 | 角色扮演后期职业 |
| **匀速增长型** | 每级增量恒定 | 稳定可预期 | 经典RPG |
| **S型曲线** | 慢→快→慢 | 自然成长感 | 大多数游戏的最佳选择 |

**等级压制公式**：
```
实际伤害 = 基础伤害 × (1 - 等级差 × 压制系数)
压制效果 = 等级差 × 每级压制百分比
```
- 等级压制强度需控制在合理范围（通常每级±2%~5%）
- 过强的等级压制会导致低等级玩家无法参与

### 战力计算公式（来自《平衡掌控者：游戏数值战斗设计》）

**通用战力公式**：
```
战力 = Σ(属性i × 权重i)
```
其中权重由属性的实际战斗贡献决定。

**设计用途**：
- 匹配系统：将战力相近的玩家匹配在一起
- AI难度评估：根据玩家战力动态调整AI强度
- 副本准入：作为关卡开放的门槛条件

**局限性**：
- 战力只是近似值，无法反映真实战斗的全部维度
- 存在「战力欺诈」问题（高战力低实战或反之）
- 不能替代实际战斗测试

### 平衡调整方法论（来自《平衡掌控者：游戏数值战斗设计》）

**数据驱动迭代循环**：
```
收集数据 → 分析问题 → 制定方案 → 调整数值 → 验证效果 → 循环
```

**4条黄金法则**：
1. **小幅调整**：每次调整幅度控制在5%-10%，避免矫枉过正
2. **单次单变量**：每次只改一个变量，确认因果后再改下一个
3. **数据验证**：每次调整后用数据验证效果，不凭感觉判断
4. **考虑生态**：一个调整必然影响其他系统，需全局评估

**平衡分析指标**：
| 指标 | 用途 | 健康范围 |
|------|------|---------|
| **胜率** | 角色的胜负表现 | 45%-55% |
| **出场率** | 角色的受欢迎程度 | >5%（确保多样性） |
| **KDA** | 击杀/死亡/助攻比 | 根据游戏类型设定 |
| **方差** | 表现的稳定性 | 越小越平衡 |

### 经济系统（来自《平衡掌控者：游戏数值战斗设计》）

**多货币体系**：

| 货币类型 | 特征 | 用途 |
|---------|------|------|
| **基础货币** | 大量产出，广泛使用 | 日常消耗品、基础装备 |
| **高级货币** | 限量产出，高价值 | 稀有道具、加速 |
| **代币** | 活动/副本产出 | 兑换限定内容 |
| **材料** | 收集合成 | 制造/升级系统 |

**5种通货膨胀控制手段**：
1. **货币回收机制**：修理、手续费、税收等消耗货币的出口
2. **产出上限**：每日/每周获取上限
3. **递进成本**：同一操作成本递增（如强化费用越来越高）
4. **限时经济活动**：节日活动后货币失效或折扣结束
5. **多货币隔离**：不同货币互不流通，避免单一货币泛滥

**资源投放节奏**：
- 前期：资源充足，加速玩家上手
- 中期：资源紧张，引导玩家做选择取舍
- 后期：资源盈余，追求极限和收藏

### 概率与随机设计（来自《平衡掌控者：游戏数值战斗设计》）

**4条核心原则**：
1. **期望公平**：玩家的实际体验应接近数学期望
2. **方差可控**：减少极端运气事件对体验的影响
3. **透明可信**：公开概率让玩家建立信任
4. **惩罚有度**：失败不应让玩家感到被系统针对

**抽卡模型**（以1% SSR概率为例）：
```
单次概率：1%
期望次数：100抽
保底机制：70抽必出SSR
综合概率：约1.43%（含保底）
```

**强化失败惩罚类型**：
| 惩罚类型 | 效果 | 心理影响 | 推荐度 |
|---------|------|---------|-------|
| 无惩罚 | 失败无损失 | 轻松但缺乏紧张感 | 中 |
| 材料消耗 | 消耗强化材料 | 可接受 | 高 |
| 降级惩罚 | 强化等级下降 | 损失感强 | 慎用 |
| 破碎惩罚 | 装备消失 | 极度挫败 | 不推荐 |

**PRD伪随机算法（Pseudo-Random Distribution）**：
- 原理：每次未触发暴击/特效时，下一次触发的概率递增
- 公式：P(N) = C × N，其中C为初始概率常数，N为连续未触发次数
- 效果：暴击/特效的实际触发更均匀，减少连续触发或连续不触发的情况
- 应用：Dota/Warcraft 3中的暴击、格挡等概率事件

### 常见数值问题（来自《平衡掌控者：游戏数值战斗设计》）

| 问题 | 表现 | 解决方案 |
|------|------|---------|
| **数值膨胀** | 数值越来越大，失去意义 | 定期数据压缩 |
| **策略退化** | 一种策略明显优于其他 | 削弱最优策略或加强其他 |
| **单属性支配** | 某属性主导一切 | 增加属性间的制约关系 |
| **等级鸿沟** | 跨等级交互完全失衡 | 等级压制系数控制 |
| **天花板效应** | 达到上限后无追求 | 扩展上限或增加新维度 |
| **地板效应** | 基础值过高使成长无感 | 降低初始值或突出成长比例 |

### 数值压缩（来自《平衡掌控者：游戏数值战斗设计》）

**4种压缩方案**：
1. **等比压缩**：所有数值统一除以系数（如100万→1000）
2. **对数压缩**：用对数函数缩小大数值差距
3. **分阶段压缩**：更新版本时重置数值基数
4. **属性合并**：精简冗余属性，减少数值总量

### 正负反馈系统设计（来自《游戏为什么好玩》）

**正反馈**（PvE主导）：让获得优势的玩家获得更多优势
- 应用：连杀奖励、经验加成、装备升级
- 效果：制造爽快感和成长感
- 风险：可能导致碾压局，降低挑战性

**负反馈**（PvP主导）：让处于劣势的玩家获得补偿
- 应用：蓝龟壳、落后玩家经验加成、动态难度调整
- 效果：维持悬念，全员参与感
- 风险：可能导致优势方感到不公平

**设计原则**：
- 确定游戏类型和核心体验，选择主导反馈类型
- 混合反馈机制往往比纯粹的单一反馈更健康
- 负反馈的设计要让优势方感受不明显，避免惩罚领先者
