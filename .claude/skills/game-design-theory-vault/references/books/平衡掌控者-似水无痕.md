## 一、数值设计基础

### 1.1 属性系统设计

属性是数值体系的基石，决定了游戏的底层规则。

**属性分类体系**：

| 属性层级 | 定义 | 示例 | 设计要点 |
|----------|------|------|----------|
| **一级属性（基础）** | 角色最底层的能力值 | 力量、敏捷、智力、体质 | 通常作为分配点数的对象 |
| **二级属性（衍生）** | 由一级属性计算得出的能力 | 攻击力、防御力、生命值 | 用于战斗公式的直接计算 |
| **三级属性（战斗）** | 战斗中生效的特殊能力 | 暴击率、命中率、闪避率 | 控制战斗中的随机性 |
| **特殊属性** | 用于数值验证/限制 | 等级、经验值、声望 | 控制成长节奏 |

**属性设计基本原则**：

1. **属性有意义原则** — 每个属性必须在游戏中有明确的用途和价值，不存在「废属性」
2. **属性可感知原则** — 属性的提升必须带来可感知的效果（伤害数字变化、战斗节奏变化）
3. **属性收益递减原则** — 堆叠单一属性时，边际收益逐渐降低，鼓励均衡发展
4. **属性克制原则** — 属性之间存在克制关系，增加策略深度
5. **属性纯净原则** — 尽量避免一属性影响所有公式的「超级属性」

**属性投放节奏**：
```
前期：升级主加1-2个主属性，感受成长
中期：开始关注副属性和战斗属性，build 雏形
后期：堆叠特定属性组合，追求极限配装
```

### 1.2 公式设计原则

**公式类型及选择**：

| 公式类型 | 表达式 | 特点 | 适合场景 |
|----------|--------|------|----------|
| **线性公式** | `y = ax + b` | 简单直观，成长稳定 | 等级与基础属性、血量成长 |
| **幂函数公式** | `y = ax^k` | 前期慢后期快 | 等级与经验值需求 |
| **对数公式** | `y = a*ln(x) + b` | 前期快后期慢 | 属性收益递减 |
| **指数公式** | `y = a*b^x` | 快速增长 | 等级压制系数 |
| **混合公式** | 多段线性/分区间函数 | 灵活定制 | 复杂的成长曲线 |

**公式设计黄金法则**：
```
1. 可解释性：公式可以被玩家理解和推算
2. 可预测性：玩家能预见到加点/装备的效果
3. 可扩展性：加入新内容时公式不需要推倒重来
4. 可验证性：数值结果可以被实际测试验证
```

**伤害简化公式模板（减法公式）**：
```
最终伤害 = max(攻击力 - 防御力, 保底伤害) × 技能倍率 × 增伤系数
```

**伤害简化公式模板（乘除法公式）**：
```
最终伤害 = 攻击力 × 技能倍率 / (1 + 防御减伤率) × 增伤系数
```

**减法 vs 乘除法公式对比**：

| 维度 | 减法公式 | 乘除法公式 |
|------|----------|------------|
| 计算复杂度 | 低 | 高 |
| 数值膨胀控制 | 容易 | 难度大 |
| 策略深度 | 低（堆甲为王） | 高（多维度博弈） |
| 反馈强度 | 攻防差值敏感 | 比例敏感，更平滑 |
| 保底伤害必要性 | 必须 | 可选 |
| 典型游戏 | 早期RPG、回合制 | 现代ARPG、MOBA |

### 1.3 成长曲线设计

**成长曲线三要素**：
1. **初始值**：进入游戏时的能力起点
2. **成长速度**：每级/每次提升的量
3. **上限值**：满级/极限时的能力终点

**标准成长曲线模型**：

```
能力值
  ^                        · (上限)
  |                      ··
  |                    ··
  |                  ··
  |                ··          ← 幂函数：前期快后期缓
  |              ··
  |            ··
  |         ···
  |   ·····              ← 前期基础阶段
  |
  +------------------------------------> 等级
```

**不同体验目的的曲线选择**：
- **前期爽快型**：前期成长快速 → 幂函数指数 < 1
- **后期封顶型**：后期放缓 → 对数函数
- **线性稳定型**：始终如一 → 线性函数
- **爆发冲刺型**：后期暴增 → 幂函数指数 > 1

**等级压制公式**：
```
等级压制系数 = 1 / (1 + e^(-k * (Lv_attack - Lv_defense)))
```
其中 k 控制压制强度，e 是自然常数。

## 二、战斗数值系统

### 2.1 伤害计算全流程

一个完整的伤害计算管道：

```
原始攻击力
    ↓
× 攻击方增益/减益修正
    ↓
- 防御方护甲/防御力（减法公式）或 × 减伤率（乘除法）
    ↓
× 技能/普攻倍率
    ↓
× 暴击判定（暴击 → × 暴击伤害倍率）
    ↓
× 属性克制系数（如适用）
    ↓
× 增伤/减伤修正（天赋、装备、Buff）
    ↓
× 随机浮动（通常 ±5%~±15%）
    ↓
= 最终伤害
```

**浮动随机区间的设计**：
- 浮动范围越小，战斗可预测性越强（策略游戏倾向小浮动）
- 浮动范围越大，战斗戏剧性越强（派对游戏倾向大浮动）
- 标准范围：±5%~±10%
- 注意：浮动不能导致「本应杀死却没杀死」的情况过于频繁

### 2.2 暴击机制

**暴击的核心数值**：
- **暴击率**：触发暴击的概率（通常 0%~100%）
- **暴击伤害倍率**：暴击时的额外伤害比例（通常 150%~300%）
- **暴击期望公式**：`E = 1 + 暴击率 × (暴击伤害倍率 - 1)`

**暴击的设计目的**：
1. 增加战斗的意外性和爽感
2. 提供一种离散的数值成长维度
3. 创造「爆发型」和「稳定型」两种 build 路线

**暴击堆叠的边际收益控制**：
```python
# 暴击率实际转化函数示例
实际暴击率 = 1 - (1 - 基础暴击率) × (1 - 装备暴击率_1) × ...
# 通过乘法叠加让边际收益递减，防止暴击率爆表
```

**暴击的三种常见实现**：

| 实现方式 | 算法 | 特点 | 案例 |
|----------|------|------|------|
| **完全随机** | 每次攻击独立判定 | 方差大，可能连续暴击或不暴 | 经典 RPG |
| **伪随机（PRD）** | 每次未暴击提高下次概率 | 更稳定，体验更公平 | Dota/War3 |
| **保底机制** | N 次不暴则必定暴击 | 下限有保障 | 手游常见方案 |

### 2.3 闪避与命中

**命中公式体系**：

标准命中判定流程：
```
实际命中率 = min(命中率 - 闪避率, 上限)
          或
实际命中率 = min(命中率 / (命中率 + 闪避率), 上限)
```

**命中率软上限**：
- 通常 80%~95% 为命中率上限（永远有 5%~20% 可能 miss）
- 命中率下限通常 5%~30%（即便满闪避也有机会被打中）
- 极端情况（100% 命中/闪避）要谨慎处理，避免公式失效

**关于「未命中」的体验设计**：
- 未命中的惩罚不能过高（尤其在连招/大招里）
- 高额资源消耗的技能不适合附加命中判定
- 命中和闪避的动画反馈要明确清晰

### 2.4 穿透与防御

**穿透机制**：
```
有效防御 = max(防御力 - 穿透值, 0)
          或
有效防御 = 防御力 × (1 - 穿透百分比)
```

**防御的收益曲线设计**：

| 设计模式 | 说明 | 效果 |
|----------|------|------|
| **线性收益** | 每点防御减少固定伤害 | 简单，但越堆越强 |
| **递减收益** | 防御越高，每点收益越低 | 平衡，鼓励多元发展 |
| **百分减伤** | 防御转化为减伤率（如 d/(d+C)） | 常见，有上限自约束 |

**减伤率标准公式**（类似 LOL/WoW）：
```
减伤率 = 防御力 / (防御力 + 常数C)
```
其中 C 为设计常数，控制防御的效率。

### 2.5 属性克制循环

**经典克制链**（三元素循环）：
```
战士 → 刺客 → 法师 → 战士
（高防克高爆） （高爆克远程） （远程克高防）
```

**克制系数设计**：
- 标准克制倍率：150%（被克制方伤害降低至 66%）
- 强克制倍率：200%（被克制方伤害降低至 50%）
- 克制程度决定了策略的重要性：克制越强，counter pick 越重要

## 三、角色成长与平衡

### 3.1 经验值曲线设计

**标准经验公式**：
```
升到 Lv.N 所需总经验 = A × N^B
```
- A：基础系数
- B：成长指数（通常 2.0~3.0）

**相对经验需求曲线**（每级所需经验）：
```
所需经验
   ^
   |                    · （后期每级要海量经验）
   |                  ··
   |                ··
   |              ··          指数增长
   |            ··
   |          ··
   |        ··
   |     ···
   |·····              （初期升级快）
   +-------------------------------->
            等级
```

**升级节奏控制**：
- 前 10 级：5-15 分钟/级，快速获得能力反馈
- 10-30 级：30-60 分钟/级，稳定成长
- 30-50 级：2-4 小时/级，形成深度体验
- 50+ 级：8+ 小时/级，荣誉象征和耐力考验

### 3.2 战力计算公式

**战力（综合战斗力）公式模板**：
```
战力 = w1 × 生命值 + w2 × 攻击力 + w3 × 防御力 + w4 × 暴击率 × 暴击伤害系数 + ...
```
- w1~w4 为权重系数，由设计者根据各属性对战斗的影响程度确定
- 战力不是绝对值，是相对参考值

**战力的用途**：
1. 关卡难度推荐
2. PvP 匹配参考
3. 玩家成长反馈
4. 数值调优的辅助工具

**战力系统的局限性**：
- 无法衡量操作水平
- 无法考虑技能组合的 synergy
- 容易被某些属性单独堆叠的情况误导

### 3.3 平衡性调整方法论

**数值调优的迭代循环**：

```
收集数据 → 分析问题 → 设计方案 → 实施调整 → 验证效果 → 继续迭代
```

**数据分析指标**：
- **胜率**：各角色/套路的胜率分布（理想 45%-55%）
- **出场率**：各角色/装备的使用频率（避免策略退化）
- **KDA 曲线**：不同熟练度下的表现差异
- **统计方差**：高方差 → 波动大不稳定，低方差 → 过于同质化

**平衡调整黄金法则**：
1. 一次只改一个变量
2. 小步微调（每次变动不超过 10%）
3. 给玩家适应时间后再评估
4. 区分「玩家觉得强」和「数据真的强」

## 四、经济系统设计

### 4.1 货币体系

**多货币设计原则**：

| 货币类型 | 来源 | 消耗 | 设计目的 |
|----------|------|------|----------|
| **基础货币（金币）** | 日常、任务、副本 | 基础消耗（药水、修理） | 日常流通、绑定在线时长 |
| **高级货币（钻石）** | 充值、活动 | 抽卡、加速、限定商品 | 营收主要来源 |
| **代币（勋章）** | 特定玩法产出 | 专属商店 | 激励特定玩法参与 |
| **材料** | 采集、掉落 | 制作、强化 | 增加资源交互深度 |

**经济系统设计目标**：
- 货币回笼速度 > 货币产出速度（控制通货膨胀）
- 多货币之间形成兑换壁垒（保护高级货币价值）
- 每种货币都有足够的消耗渠道（避免货币堆积贬值）

### 4.2 通货膨胀控制

**常见的通货膨胀原因**：
- 产出过快（过多的金币奖励）
- 消耗不足（缺乏足够的高额消耗项目）
- 复利效应（投资/交易系统的不断累积）

**控制手段**：
```
1. 税收系统（交易手续费 5%-15%）
2. 持续的新消耗内容（新养成、新商店）
3. 定期结算/重置（赛季制、月卡）
4. 非永久性消耗（修理费、传送费、Buff消耗）
5. 数量限制（每日限购、每周上限）
```

### 4.3 资源投放节奏

**F2P 游戏的资源投放设计**：
- 前期（第 1-7 天）：大量投放，让玩家快速成长
- 中期（第 2-4 周）：持续投放，但开始出现瓶颈
- 后期（1 个月后）：精确投放，付费点凸显
- 长线（3 个月后）：稳定产出，维持活跃

## 五、概率与随机设计

### 5.1 概率设计核心原则

**1. 期望公平原则**
- 玩家期望的 F(P) ≤ 实际体验的 F(P)
- 即「感觉上的概率」应略高于「实际概率」，避免玩家觉得被坑

**2. 方差控制原则**
- 高方差（如抽卡）需要保底机制
- 低方差（如强化成功）接受波动但不可过于极端

**3. 间隔控制原则**
- 连续两次随机事件之间要有冷却时间
- 避免「连败」打击玩家积极性

**4. 透明性原则**
- 真实概率应公开（中国法律要求）
- 概率描述不误导（如 50% 概率 ≠ 两次必中一次）

### 5.2 抽卡/扭蛋设计

**标准抽卡模型**：
```
SSR 概率 1% → 70 抽保底 → 50% 概率获得限定角色 → 300 抽可兑换
SR  概率 9% → 10 抽保底 → 余下为过渡角色
R   概率 90% → 每次保底 R 以上
```

**保底机制的数学原理**：
- 无保底时，期望抽取数 = 1/p（p 为概率，1% → 100 抽）
- 有保底时，期望抽取数显著降低，方差也大幅降低
- 保底值通常设为期望值的 1.5-3 倍

### 5.3 强化/锻造概率

**强化失败的设计**：
| 失败惩罚类型 | 说明 | 体验特征 |
|-------------|------|----------|
| **无惩罚** | 失败不掉级 | 轻松，鼓励尝试 |
| **掉级** | 失败掉 1-2 级 | 焦虑，谨慎决策 |
| **归零** | 失败归零从头来 | 高风险，极端情绪 |
| **破坏** | 装备消失 | 极差体验，慎用 |
| **保底值** | 失败累积进度值 | 努力看得见，接受度高 |

**强化概率模型推荐**：
```
前50%阶段：100%成功 → 建立信心
中间30%阶段：80%-50% → 逐步降低成功率
最后20%阶段：50%-10% → 高难度，但高回报
附加强化失败进度保底：失败N次后必定成功
```

### 5.4 伪随机算法（PRD）

War3 风格的 PRD 算法：

```python
def prd_attack(c, n=1):
    """
    c: 理论暴击率
    n: 已连续未暴击次数
    return: 本次实际暴击率
    """
    P = c * n
    return min(P, 1.0)
```

PRD 的优势：
- 暴击分布更均匀，不会出现极端连暴/连不暴
- 实际感知更接近理论概率
- 玩家更容易相信「概率是真实的」

## 六、数值验证与调优方法

### 6.1 数值验证手段

| 验证方法 | 操作 | 验证内容 |
|----------|------|----------|
| **静态验算** | Excel/表格人工核对 | 公式逻辑、边界值 |
| **本地测试** | 本地运行验证 | 实际表现、手感、数值反馈 |
| **批量测试** | 自动化跑大量样本 | 统计分布、期望值验证 |
| **Beta 测试** | 玩家实际体验 | 平衡性、满意度、付费意愿 |
| **全量数据** | 上线后监控 | 真实数据、异常检测 |

### 6.2 常见数值问题

| 问题 | 表现 | 原因 | 解决方法 |
|------|------|------|----------|
| **数值膨胀** | 新内容数值远超旧内容 | 公式设计缺陷、长期无收敛 | 建立数值收敛机制、属性压缩 |
| **策略退化** | 所有人用同一套最强搭配 | 某属性/组合收益过高 | 平衡各属性收益曲线 |
| **单属性支配** | 某个属性在所有场景都有效 | 属性维度设计不足 | 增加属性用途差异化 |
| **等级鸿沟** | 高一级 = 碾压 | 等级压制系数过高 | 压制系数不要超过 5%/级 |
| **天花板效应** | 满级后无事可做 | 后期内容不足 | 设计满级后的各种玩法 |
| **地板效应** | 某属性低到无用 | 属性设计未覆盖所有场景 | 给低属性提供保底机制 |

### 6.3 数值压缩（数值重置）

**数值压缩的场景**：
- 游戏运营多年后，属性从万级膨胀到亿级
- 新玩家无法追赶老玩家
- 数字过大失去可读性（伤害显示从 8421 变成 8.4k 再到 84M）

**常见压缩方案**：
```
方案1：整体除以 N（如 WoW 的属性压缩）
方案2：重置赛季，新赛季重新累积
方案3：新养成线替换旧养成线
方案4：新增数值层级（如魔化/觉醒等重新洗牌）
```

## 七、常见数值模型

### 7.1 标准 RPG 数值模型

```
等级 1-100
攻击力：10 (Lv1) → 500 (Lv100) 线性成长
生命值：100 (Lv1) → 10000 (Lv100) 幂函数成长（更快）
经验值：100 (Lv1→2) → 100000 (Lv99→100) 指数成长
装备加成：每件装备提升当前等级 10%-20% 的属性
Boss 强度：比同级普通怪物强 3-5 倍
```

### 7.2 MOBA 类数值模型

```
英雄属性成长（线性）：
- 每级提升攻击力约 2-5
- 每级提升生命值约 50-100
- 技能伤害 = 基础值 + 攻击力/法术强度 × 系数

装备性价比平衡：
- 同类装备的金币/属性效率控制在 ±10% 以内
- 前期装备效率略高（促进购买）
- 后期装备提供独特效果而非单纯堆属性

防御塔伤害：固定伤害 + 随时间递增
小兵属性：随时间线性增长，控制推线速度
```

### 7.3 卡牌/数值收集类模型

```
卡牌等级：Lv1 → Lv100（每次升级需要 N 张同名卡）
卡牌稀有度：R / SR / SSR / UR（基础属性差距 10%-20%）
进阶系统：集齐 N 张进阶一次，提升 10-15% 属性
技能升级：提升技能倍率 1%-3%/级，有等级上限

核心公式：
战斗力 = 基础属性 × 稀有度系数 × 等级系数 × 装备系数 × 技能系数
```

## 八、金句摘录

1. 「数值设计不是数学题，是心理学。同一个伤害数字，放在不同的 UI 位置，玩家的感受完全不同。」

2. 「没有完美的公式，只有最适合当前游戏的公式。减法公式简单粗暴，乘除法公式平滑可控，选哪个取决于你想要什么体验。」

3. 「玩家说『平衡性差』的时候，往往不是指数值不对，而是『凭什么他能赢我』的不公平感。」

4. 「一个优秀的数值策划，70% 的时间在分析数据，20% 在调整，10% 才是在写公式。」

5. 「不要为了调整平衡性而毁了游戏特色。不平衡的魅力比平庸的平衡更有价值。」

6. 「数值膨胀不是问题，失控的数值膨胀才是。设计之初就要留好压缩空间。」

7. 「好装备让人变强10%，坏装备让人变弱50%。后者的设计比前者更重要——因为玩家对失去的恐惧远大于对得到的渴望。」

8. 「伪随机算法不是为了骗玩家，而是为了让『公平体验』接近『理论公平』。」

9. 「所有概率系统的最后一道防线是保底——玩家可以接受脸黑，但不能接受无底洞。」

10. 「战力和推荐等级只是参考线，顶尖玩家永远能用低战力通关——但你不能为了让高手有挑战而让普通玩家卡关。」

11. 「游戏中所有的随机都应该可控制、可影响、可规避。完全不可控的随机是对玩家努力的不尊重。」

12. 「属性越多，设计越自由，但平衡越难。新手设计师喜欢加属性，老手设计师喜欢删属性。」
